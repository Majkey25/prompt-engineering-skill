# Contributing

Thanks for contributing to this skill.

## Guidelines

- Open an issue first for large or structural changes.
- Keep instructions practical and directly useful for real agent workflows.
- Avoid vague prompt engineering theory with no operational value.
- Add realistic examples when changing behavior, templates, or prompt patterns.
- Keep compatibility with ChatGPT, Codex, Claude, Claude Code, and similar agent workflows where possible.

## Pull requests

1. Fork the repo and create a branch.
2. Make focused changes with clear commit messages.
3. Update documentation when behavior changes.
4. Submit a pull request with a short summary of what changed and why.
